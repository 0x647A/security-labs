# Glossary of DFIR Terms and Acronyms

## A
- **APT** – Advanced Persistent Threat
- **AIDE** – Advanced Intrusion Detection Environment
- **Attribution** – The process of identifying perpetrators of a cyber attack

## B
- **Business Continuity** – The ability of an organization to continue critical functions during an incident

## C
- **C&C/C2** – Command and Control - Infrastructure controlled by attackers to manage malware
- **Chain of Custody** – Documented path of control over digital evidence
- **Chain of Evidence** – Documented history of each piece of evidence from collection to presentation
- **Containment** – Actions taken to stop the spread of an incident

## D
- **DFIR** – Digital Forensics and Incident Response - The practice of investigating digital incidents and analyzing evidence
- **Data Exfiltration** – Unauthorized removal of data from an organization
- **DWARF** – Debugging With Attributed Record Formats - Kernel symbol format for Volatility

## E
- **EDR** – Endpoint Detection and Response - Technology for detecting and responding to threats on endpoints

## F
- **Full Packet Capture** – Complete capture of network packets for later analysis

## H
- **Hash Values** – Cryptographic hash values (MD5, SHA-1, SHA-256) used to verify data integrity
- **HIDS/HIPS** – Host-based Intrusion Detection/Prevention System - System for detecting or preventing intrusions on individual hosts

## I
- **IDS/IPS** – Intrusion Detection System / Intrusion Prevention System - Systems for detecting or preventing network intrusions
- **IoC** – Indicators of Compromise - Artifacts that indicate a system has been compromised
- **ISF** – Intermediate Symbol Format - Symbol format for memory analysis in Volatility

## K
- **Kill Chain** – Model describing the stages of a cyber attack: reconnaissance, weaponization, delivery, exploitation, installation, command and control, actions on objectives

## L
- **Lateral Movement** – Movement through a network after initial compromise

## M
- **Malware** – Malicious Software - Harmful programs such as viruses, trojans, spyware, and adware
- **Memory Dump** – A complete copy of RAM contents containing processes, credentials, and cryptographic keys
- **MITRE ATT&CK** – Framework describing tactics and techniques used by attackers

## N
- **NIST SP 800-86** – NIST standard for computer forensics and digital investigations

## O
- **OSQuery** – Framework for querying the operating system using SQL syntax

## P
- **PCAP** – Packet Capture - File format for storing captured network packets
- **Phishing** – Social engineering attack technique involving impersonation of trusted entities
- **Privilege Escalation** – Gaining elevated permissions on a system

## R
- **Ransomware** – Malicious software that encrypts user files to extort payment
- **RFC 3227** – RFC document providing guidelines for collecting and archiving digital evidence
- **Risk Assessment** – Process of identifying, analyzing, and evaluating threats
- **Rootkit** – Malicious software that hides its presence by modifying core system functions

## S
- **SIEM** – Security Information and Event Management - System for aggregating and analyzing security data and events
- **Steganography** – Technique for hiding information within other files or data

## T
- **Timeline Analysis** – Chronological analysis of events to reconstruct the sequence of incidents
- **TTPs** – Tactics, Techniques, and Procedures - Methods and approaches used by attackers
- **TSURUGI Linux** – Linux distribution dedicated to forensic analysis with preinstalled DFIR tools

## V
- **Volatility** – Framework for analyzing RAM dumps, extracting processes and artifacts

## W
- **Write Blocker** – Device or software that prevents data modification during forensic analysis
- **Wireshark** – Network protocol analyzer for capturing and analyzing traffic

## Y
- **YARA** – Yet Another Recursive Acronym - Tool for identifying and classifying malware based on patterns

## Z
- **Zeek** – Network traffic analysis platform (formerly Bro) that generates detailed network communication logs

## Standards and Frameworks
- **ISO/IEC 27035** – International standard for information security incident management
- **ISO/IEC 27037** – Standard for identification, collection, and preservation of digital evidence
- **NIST Incident Response Cycle** – Incident response phases: Preparation → Detection & Analysis → Containment, Eradication & Recovery → Post-Incident Activity

## Key Tools
- **Autopsy** – Graphical platform for comprehensive forensic analysis
- **Security Onion** – Linux distribution integrating IDS, traffic analysis, and SIEM
- **Sleuth Kit** – Open-source toolkit for analyzing file system evidence
- **tcpdump/tshark** – Command-line tools for capturing and analyzing network traffic
