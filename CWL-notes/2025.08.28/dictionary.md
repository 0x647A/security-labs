### A  
- **ACL** – Access Control List: defines permissions for resources  
- **AWS** – Amazon Web Services: cloud services platform by Amazon  

### B  
- **Bastion Host** – a hardened server providing secure access to private networks  

### C  
- **CVE** – Common Vulnerabilities and Exposures: public database of known security flaws  
- **Certificate Transparency** – logging system for SSL/TLS certificates  
- **CLI** – Command Line Interface: text-based interface for managing services  
- **Cloud Native** – applications designed specifically for cloud environments  

### D  
- **Decoy Scan** – using spoofed IP addresses to mask scanning origin  
- **Directory Fuzzing** – brute-forcing web directories and files  

### E  
- **Egress Filtering** – controlling outgoing network traffic  

### F  
- **Fingerprinting** – identifying software versions and types  

### G  
- **GCP** – Google Cloud Platform: cloud services platform by Google  
- **Google Dorking** – using advanced Google search operators for reconnaissance  
- **gowitness** – automated website screenshot and reporting tool  
- **grayhatwarfare** – search engine for public cloud storage  

### I  
- **IAM** – Identity and Access Management: controlling user access and roles  
- **IMDS** – Instance Metadata Service: retrieves VM metadata including temporary credentials  
- **Ingress** – incoming network traffic  

### J  
- **Jump Box** – intermediate host used to access private resources  

### L  
- **Least Privilege** – granting only the minimal necessary permissions  

### M  
- **Magic Numbers** – numeric IP representations (hex, octal, decimal) for bypassing filters  
- **masscan** – very fast port scanner  

### N  
- **NACL** – Network Access Control List: subnet-level traffic rules  
- **NAT** – Network Address Translation: translates private to public IPs  
- **nip.io** – wildcard DNS resolving subdomains to IPs  
- **NSE** – Nmap Scripting Engine: extends nmap with custom scripts  
- **Network Enumeration** – discovering network hosts and services  

### O  
- **OSINT** – Open Source Intelligence: gathering intelligence from public sources  

### R  
- **REST** – Representational State Transfer: API architectural style using HTTP  
- **RCE** – Remote Code Execution: executing arbitrary code on a remote system  

### S  
- **SG** – Security Group: instance-level firewall  
- **SOCKS** – Socket Secure: proxy protocol for tunneling traffic  
- **SSRF** – Server-Side Request Forgery: forcing a server to make unintended HTTP requests  
- **Shared Responsibility Model** – delineation of cloud security duties between provider and customer  
- **SRI** – Subresource Integrity: ensures integrity of CDN-loaded resources  
- **Subdomain Enumeration** – discovering DNS subdomains  

### T  
- **Token-based Authentication** – access control using security tokens  

### U  
- **User-data Script** – initialization script run on VM startup  

### V  
- **VPC** – Virtual Private Cloud: isolated virtual network in the cloud  
- **Virtual Network** – software-defined network in cloud environments  

### W  
- **WAF** – Web Application Firewall: filters and monitors HTTP traffic  
- **webhook.site** – service for capturing HTTP webhooks  

### X  
- **XSS** – Cross-Site Scripting: injecting client-side scripts into web pages  
- **XXE** – XML External Entity: exploiting XML parsers to access external resources  

### Z  
- **Zero Trust** – security model assuming no implicit trust, verifying every request
